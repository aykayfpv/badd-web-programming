# Aufgaben zu UI-Frameworks

Die beste Übung ist das praktische Arbeiten mit den Frameworks. Setzen Sie
verschiedene Projekte mit verschiedenen Frameworks Ihrer Wahl um! Diskutieren
Sie gemeinsam über Ihre Erfahrungen im Umgang mit den Frameworks!

Einige Frameworks sind beispielsweise:

* Vanilla.js, d.h. nur native Funktionen und __kein__ Framework
* [JQuery](https://jquery.com/)
* [Knockout](http://knockoutjs.com/)
* [Elm](https://elm-lang.org/)
* [Vue](https://vuejs.org/)
* [React](https://react.dev/)
* [Angular](https://angular.dev/)
* [Svelte](https://svelte.dev/)
* [Custom elements (web components)](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements)

## Aufgabe 1 - Hallo Welt

Setzen Sie eine Hallo-Welt-Anwendung mit verschiedenen Frameworks Ihres Vertrauens um!

Eine Hallo-Welt-Anwendung besteht im Wesentlichen daraus, dass der Text `Hallo Welt`
ausgegeben wird.

Solch eine Anwendung dient dazu, erst einmal das Framework aufzusetzen und sicherzustellen,
dass man bereit für die weitere Entwicklung ist.

## Aufgabe 2 - Zähler

Setzen Sie eine Zähler-Anwendung mit verschiedenen Frameworks Ihres Vertrauens um!

Eine Zähler-Anwendung besteht aus der Anzeige von 2 Zähler-Werten, die über Buttons
erhöht, erniedrigt oder zurückgesetzt werden kann. Zudem soll unter den beiden
Zählern immer noch die aktuelle Summe beider Zähler angezeigt werden. 

Solch eine Anwendung dient dazu, herauszufinden, wie verschiedene Frameworks
mit Nutzerinteraktion, dem dynamischen Aktualisieren der UI und abgeleiteten Werten
umgehen.

## Aufgabe 3 - Formular

Setzen Sie eine Kontaktdatenformular mit verschiedenen Frameworks Ihres Vertrauens um!

Eine Kontaktdatenformular besteht aus verschiedenen Eingabeelementen für z.B.
Vorname, Nachname, Alter, E-Mail usw.; sowie einem Button zum Absenden. Zudem
sollten Eingabe sinnvoll validiert werden (Mussfelder, Datentyp).

Solch eine Kontaktdatenformular dient dazu, herauszufinden, wie verschiedene Frameworks
mit nativen HTML-Formularelementen umgehen bzw. welche andere Möglichkeiten
für Formulare sie bieten.

## Aufgabe 4 - Todo

Setzen Sie eine Todo-App mit verschiedenen Frameworks Ihres Vertrauens um!

Siehe auch [TodoMVC](https://todomvc.com/), [Demo](https://todomvc.com/examples/javascript-es6/dist/),
[App specification](https://github.com/tastejs/todomvc/blob/master/app-spec.md)

Solch eine Todo-App dient dazu, herauszufinden, wie verschiedene Frameworks
mit leicht komplexerer Interaktivität umgehen und wie einfach verschiedene
Komponenten erstellt werden können.

## Aufgabe 5 - REST-API-Anbindung

Binden Sie eine REST-API mit verschiedenen Frameworks Ihres Vertrauens an!

Beispielsweise könnten Sie 2 Eingabefelder für die Höhe und Breite in Pixeln erstellen
und mittels des [image-Endpunkts](https://dummyjson.com/docs/image) von `dummyjson.com`
nach Knopfdruck ein Bild in dieser Größe dynamisch laden und anzeigen.

Implementieren Sie auch entsprechende Ladeanimationen und testen Sie diese
durch Reduzierung der Netzwerkgeschwindigkeit in den Dev-Tools!

Solch eine REST-API-Anbindung dient dazu, herauszufinden, wie verschiedene Frameworks
mit asynchronen Daten und Operationen umgehen.
