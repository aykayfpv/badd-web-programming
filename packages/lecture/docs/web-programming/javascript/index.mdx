# Kapitel 3 - JavaScript und TypeScript

In diesem Kapitel schauen wir uns die Programmiersprache JavaScript an.

Mit JavaScript können wir Webseiten dynamisch gestalten. Nahezu alle modernen
Webseiten verwenden JavaScript in der einen oder anderen Form und ist kaum
noch wegzudenken.

- [Wikipedia](https://www.wikipedia.org/)
  - Optional, aber Zusatzfunktionen wie Bildvorschau im Overlay
- Webshops
- Soziale Netzwerke wie Facebook, Twitter, Reddit, Bluesky
  - UI wird teilweise erst durch den Browser erzeugt. Meta (Facebook) war etwa
    beteiltigt an der UI-Bibliothek  `react`.
- Videoplattformen wie Odysee oder YouTube
- Anwendungen wie [draw.io](https://app.diagrams.net/) (Diagramme zeichnen)
  oder Pixlr (Bildbearbeitung).

JavaScript wird offiziell bezeichnet als [EcmaScript](https://github.com/getify/You-Dont-Know-JS/blob/2nd-ed/get-started/ch1.md).

EcmaScript wird betreut von [ecma iInternational](https://ecma-international.org/history/),
welche jedes Jahr eine neue Version der Programmiersprache herausbringen. Die
Spezifikation selber kann hier eingesehen werden: [ecma262](https://tc39.es/ecma262/).

[TC39](https://tc39.es/) ist eine von ecma International organsisierte Gruppe
von Fachleuten, welche sich um die Weiterentwicklung der Programmiersprache
kümmern. Neue Features für EcmaScript heißen [Proposals](https://github.com/tc39/proposals),
welche durch verschiedene [Stages](https://tc39.es/process-document/) laufen. Schafft
ein Feature es in die letzte Stage 4, wird es offiziell in den EcmaScript-Standard
aufgenommen. Einige nennenswerte Proposals aktuell sind etwa:

- [Temporal-API](https://github.com/tc39/proposal-temporal)
- [Decorators](https://github.com/tc39/proposal-decorators)
- [Shadow Realm](https://github.com/tc39/proposal-shadowrealm) (Hat nichts mit einem Kartenspiel zu tun!)
- [Type-Annotations](https://github.com/tc39/proposal-type-annotations)
- [Pipeline-Operator](https://github.com/tc39/proposal-pipeline-operator)
